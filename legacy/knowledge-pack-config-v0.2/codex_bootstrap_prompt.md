# 首次交给 Codex 的任务

请将本仓库初始化为“丰行慧运互联网情报研究工作区”。

## 核心要求

- 只研究丰行慧运，不与丰图情报混合。
- 完整读取 `knowledge/` 下的知识包。
- 以 `05_strategy_2026.md` 作为当前战略的最高优先级。
- 历史丰图交通货运资料只能作为能力背景，不得自动归属丰行。
- 主动在中英文公开互联网广泛搜索，不等待我提供新闻。

## 第一阶段：知识校验

1. 根据知识包整理丰行的定位、业务域、产品、客户、场景、政策主题和技术主题。
2. 检查 `07_watchlist.yaml`、`08_query_library.yaml` 和 `09_terms_aliases.yaml` 是否一致。
3. 将无法确认的问题汇总到 `output/pending-confirmation.md`。
4. 不得自行填补公司事实。

## 第二阶段：最近 7 天试运行

覆盖：

1. 政策与监管动态
2. 竞品与标杆企业动态
3. 产业与市场趋势
4. 客户需求与场景机会
5. 产品与技术动态

重点围绕：货主运输数字化、车队经营与安全、政府道路运输监管、营运车险、运输能耗与车后、新能源/智驾/无人货运、交通数据资产。

## 搜索与核验

- 同时进行中文和英文查询。
- 每类至少使用 5 组不同查询思路。
- 优先政府、企业官网、公告、采购、财报、产品文档和正式研究报告。
- 必须打开正式候选的原始页面，确认日期和关键事实。
- 产业趋势需要两个独立来源，或一份高可信研究报告。
- 同一事件的转载必须合并。

## 输出

生成：

- `candidate-pool.md`
- `source-audit.md`
- `analyst-review.md`
- `selected-intelligence.json`
- `google-ai-studio-input.md`

每条候选必须标明：

- 对应丰行业务域
- 对应客户/产品/场景
- 来源事实
- 分析推演
- 对丰行的启示
- 建议动作
- 不确定性
- 评分与入选理由

不得修改网站，不得自动发布。
