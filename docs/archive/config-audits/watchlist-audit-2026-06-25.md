# Watchlist Audit

## 本次处理范围

本次只进行 `config/watchlist.yaml` 的实体清洗和可执行性升级，并同步更新 `config/entity-aliases.yaml`、检查并调整 `config/query-library.yaml` 的结构一致性。

本次未执行互联网情报搜索，未核验实时官网、工商信息、公告、财报或产品文档，因此没有把任何新增信息标记为 `verified`。

- 所有实体的 `official_sources` 均未在本轮实时核验，统一保持空数组。
- 多数企业的法律主体、英文名、产品正式名称只来自现有项目资料或旧配置，未经过官网、工商、公告或 IR 材料核验，因此 `verification.status` 设为 `partial` 或 `pending`。

## 地方北斗运营商的发现规则

地方北斗运营商不是单一企业。已在 `watchlist.yaml` 中建立 `discovery_rules.local_beidou_operators`，要求按省市组合搜索：

- 省份/城市 + 北斗运营商 + 道路运输动态监管
- 省份/城市 + 北斗车辆监控 + 两客一危一重
- 省份/城市 + 重载普货 + 主动安全 + 车辆监控平台
- 政府采购平台、招投标平台和中标公告中的平台建设、平台运维、运营服务商、联合体成员信息

新发现主体必须记录企业名称、所在省市、来源 URL、来源等级、主要客户或采购方、监管项目、政府关系、项目规模或车辆类型、是否涉及主动安全、保险或货运企业服务、与丰行重叠市场、建议关系类型和待确认事项。

## 新竞品加入观察名单的规则

“其他后续发现的丰行竞品”已改为 `discovery_rules.additional_competitors`。新企业必须满足：

- 已打开来源正文并确认企业名称、产品或项目、日期和来源等级。
- 能映射到丰行至少一个核心业务域、产品能力、客户类型或场景。
- 有 A/B 级来源，或至少两次高相关事件；C 级来源只能作为线索。
- 能说明核心产品、主要客户、与丰行重叠市场、建议竞争关系、判断依据和来源 URL。
- 未经人工确认，不得列为 `direct_competitor`。
