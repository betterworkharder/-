# 来源规则

## 基本原则

正式情报必须打开并阅读来源正文，不得依据搜索摘要、标题或转载片段生成。每条正式候选必须记录来源名称、原始 URL、发布时间、事件发生时间、是否原始来源、是否转载、是否支持标题和摘要、是否存在矛盾信息以及是否需要第二来源核验。

情报中心成品以新闻、行业资讯、政策公告、企业动态、产品更新、招投标和市场事件为主。论文、预印本、arXiv、学术会议论文和纯学术研究不得进入 Google AI Studio 成品文本或情报中心正式栏目；它们只能作为后台研究、技术理解或概念解释的参考材料。除非用户明确要求“论文扫描”或“技术论文专栏”，不得把论文类内容写成正式情报条目。

C 级来源只能作为线索，不能单独支撑正式情报。重大数字、客户、项目规模、市场与客户趋势、技术与能力演进中的趋势判断，原则上需要 A 级来源，或两个相互独立的可靠来源。

正式周度研究必须先扫描 `config/source-index.yaml` 中的固定来源，再进行开放搜索。对竞合与标杆动向，必须逐一对固定竞品和观察名单企业执行东方财富网、新浪网/新浪财经站内检索，并对泰伯网进行地理信息、时空智能、地图、智慧交通、车路协同和交通数字化相关检索。

东方财富网可用于发现财报、公告、投资者互动、资本市场报道和上市公司动态；新浪网/新浪财经可用于发现财报、公告、资本市场报道、企业动态、行业报道和舆情线索；泰伯网可用于发现地理信息、时空智能、地图、智慧交通和相关企业动态线索。东方财富网和新浪网/新浪财经可以作为正式候选来源：正文具备明确来源、发布时间、可核验事实和业务相关性时，可按 B 级或 B/C 级纳入候选。重大数字、客户、项目规模、财务指标或监管事项仍应优先回溯企业公告、企业官网、投资者关系材料、巨潮资讯、SEC/港交所/交易所公告、政府采购/招投标原文或其他 A/B 级来源；无法回溯但正文证据较完整时，必须在来源审计中标注不确定性。

## 泰伯数据强制扫描规则

泰伯相关扫描必须同时覆盖 `taibo.cn` 公开网页和 `data.taibo.cn` 情报数据库。`data.taibo.cn` 是前端单页应用，详情页正文通常不在静态 HTML 中，必须优先使用以下 API 读取结构化信息：

- 最新监测：`https://data.taibo.cn/api/info/index?page={page}`
- 产业动态：`https://data.taibo.cn/api/info/index?type=0&page={page}`
- 市场情报：`https://data.taibo.cn/api/info/index?type=1&page={page}`
- 情报详情：`https://data.taibo.cn/api/info/view?id={id}`

每次周度研究至少扫描最近 7 天的最新监测、产业动态和市场情报列表页。对命中丰行直接业务、邻近行业能力或重要政策/项目/资本信号的条目，必须打开详情 API，读取标题、来源、发布时间、摘要、正文、关联企业和原始 URL 字段。

泰伯数据条目按三层处理：

- 直接相关：货运、物流、商用车、车队、道路运输、运输监管、车辆监管、智能调度、低空物流、智能网联货车、自动驾驶货运等，必须进入候选池并回溯原始来源。
- 邻近相关：时空智能、空间数据、数据空间、智慧交通、车路协同、智能网联汽车、自动驾驶仿真、数据合成、场景库、物理世界模型、空间感知、遥感、卫星数据等，至少进入观察池或淘汰留痕。
- 背景观察：具身智能、机器人、通用世界模型、通用大模型、低空经济、商业航天等，如与丰行直接业务暂不明确，可以淘汰或低相关观察，但不得静默丢弃。

未执行泰伯数据 API 扫描时，不得在 `candidate-pool.md` 或最终说明中写“泰伯网未发现有效新增信息”；只能写“泰伯数据 API 固定来源未完成扫描”。执行扫描后，`candidate-pool.md` 必须保留泰伯数据扫描摘要，包括扫描页数、发现条目数、命中条目数、入选/观察/淘汰数量和典型淘汰原因。

泰伯数据原则上仍是线索或补充来源。正式候选应继续回溯政府原文、企业官网/官微、招投标原文、上市公司公告、投资者关系材料或其他 A/B 级来源。无法回溯原始来源时，除非泰伯正文和来源信息已足够可靠且来源等级符合 B 级要求，否则进入观察池或标注“发现信息但无法核验”。

## 周报模块去重规则

同一期不同模块不能重复同一新闻、链接、官方原文、企业动态或事实事件；跨历史周报也不能重复已报告事件。同一信息只能放进最匹配的一个模块。

周报研究必须覆盖 5 个固定模块：政策趋势与监管、资金与项目机会、竞合与标杆动向、市场与客户趋势、技术与能力演进。覆盖不等于强行收录；如果某一模块没有达到正式候选标准的新情报，应明确写“本期该栏目未发现达到正式候选标准的新情报”，并在候选池中记录扫描摘要、观察池或淘汰原因。

模块分类优先级按事实事件本身判断，而不是按来源网站判断：

- 政策、制度、监管和合规边界优先进入政策趋势与监管。
- 采购、招投标、奖补、试点、示范工程、重点工程和客户侧建设任务优先进入资金与项目机会。
- 多来源共同证明的客户需求变化、行业需求变化和市场共性变化进入市场与客户趋势。
- 单家公司或单一主体的产品、合作、融资、财报、人事、客户落地等动作进入竞合与标杆动向。
- 跨公司、跨行业或由标准/官方文档/高质量报告支撑的能力范式变化进入技术与能力演进；单家公司产品发布原则上不放入技术与能力演进。

## 资金与项目机会来源规则

资金与项目机会栏目优先使用中文网站原文。除非用户明确要求海外案例，否则该栏目不得使用英文网站作为正式来源。

资金与项目机会只收录奖补、采购、招投标、试点、示范工程、重点工程、客户侧建设任务等明确落地机会。没有具体资金、项目、采购、试点或建设动作的信息，不得放入本模块。

资金与项目机会栏目不得使用以下来源作为正式候选：

- 已在竞合与标杆动向栏目使用的企业动态；
- 已在政策趋势与监管栏目使用的政策新闻；
- 已在技术与能力演进、市场与客户趋势等其他模块使用的同一事件；
- 仅有搜索摘要、转载片段或无法打开正文的链接。

资金与项目机会栏目必须逐条打开原文链接并阅读正文后再写。正文必须能够支撑机会类型、牵头主体、涉及地区/行业和可跟进点；不得只根据标题、关键词或外部摘要推断商机。

同一周度文本中不得出现重复新闻链接。同一事件只能进入一个主要栏目；如果事件更像竞合动向、政策监管、市场趋势或技术演进，应放入对应栏目，不得为了填充资金与项目机会栏目重复使用。

资金与项目机会栏目应围绕丰图可合理触达的项目机会展开，包括地址治理、地图底座、企业核验、路径规划、城市运行、数据治理，以及与丰行慧运相关的货主运输管理、车队运营、安全风控、能源补能、园区/场站协同、保险风控、运输监管、海外运输和供应链协同等方向。丰图相关能力只能作为可能支撑，不得写成丰行已具备或已商业化的能力，除非知识包已有明确依据。

## A级来源

A级来源是可直接支撑正式情报的优先来源：

- 政府和监管机构官网
- 正式政策法规
- 企业官网
- 上市公司公告和财报
- 投资者关系材料
- 官方招投标和政府采购平台
- 产品官方文档
- 官方新闻稿和官方白皮书
- 高校、研究机构、咨询机构发布的行业报告或白皮书

## B级来源

B级来源可用于补充事实、交叉核验和解释背景；重要结论应尽量回溯 A 级来源：

- 权威媒体
- 行业协会
- 知名研究机构
- 专业行业媒体
- 高质量专访
- 具有明确采访对象和原始出处的行业报道
- 泰伯网等有明确采编来源、发布时间和正文证据的垂直行业媒体报道
- 东方财富网、新浪网/新浪财经等财经媒体中有明确公告、研报、互动易、上市公司材料出处，或有清晰采编来源、发布时间和正文证据的报道

## C级来源

C级来源只能作为线索，不得单独支撑正式情报：

- 自媒体
- 聚合网站
- 社交媒体
- 搜索摘要
- 无出处转载
- 论坛、个人文章和未经核验的二次整理
- 东方财富股吧、行情快讯、无公告出处的个股短讯
- 新浪股吧、无明确来源的滚动快讯、无法打开正文或无法确认发布时间的转载短讯

## 淘汰规则

以下情况不得进入正式情报：没有明确来源、无法确认发布时间、未打开正文、正文无法支撑标题或摘要、与丰行业务无合理联系、仅与丰图其他业务相关、论文/预印本/arXiv/学术会议论文作为成品条目、旧闻重新包装、重复转载、纯观点无新增事实、标题相关正文无关、只读搜索摘要、来源疑似虚假、关键数据无法核验、把规划能力写成已上线能力、把历史项目写成当前业务、把市场趋势写成明确商机、把合作可能性写成已达成合作、把模型推演写成来源事实。
