# 03 产品与能力矩阵

## 1. 货主与供应链数字化

### 核心产品/场景

- 运输 OTMS
- 生产制造 OTMS 场景套件
- 货运地图数字化决策套件
- 网络货运与合规风控
- 最优线路规划
- 资源结构规划
- 智能排线
- 动态资源调度
- 在途数字化监控
- 园区/场所数字化
- 结算对账与履约分析

### 关键能力

- 仓网与线路规划
- 多式联运与配载排线
- 运力资源规划、画像与匹配
- 静态计划 + 动态补偿
- 实时货量驱动的动态路由
- 货车禁限行、里程、路桥费、时效与成本模型
- 车辆实时位置、轨迹回放、路线偏离与 ETA
- 地图 + AIoT 的远程指挥调度
- 地图 + 视觉算法的园区调度和安全作业管理
- 对账、结算、异常和履约分析

### 货主产品链路理解

货主侧产品应按“规划-调度-执行-结算-优化”理解：

1. **规划**：仓网、线路、车型、时窗、里程、路桥费、成本、禁限行、服务半径。
2. **调度**：资源池、自有/外协运力、临时找车、车货匹配、静态计划、动态补偿。
3. **执行**：派车、到离场、在途位置、ETA、偏航、超时、异常、回单。
4. **结算**：里程核验、路桥费、油耗/电耗、承运商绩效、异常责任。
5. **优化**：线路复盘、承运商优化、资源结构重算、成本/时效/安全多目标平衡。

## 2. 车队数智化

### 产品能力分组

1. **安全产品**
   - ADAS/DMS 风险识别
   - 驾驶行为分析
   - 风险画像和分层
   - 7×24 安全干预
   - 培训、整改、复盘与闭环运营

2. **履约产品**
   - 任务、派车、在途和时效管理
   - ETA、路线偏离和异常处理
   - 运营报表和绩效分析

3. **能耗产品**
   - 油耗/电耗分析
   - 补能规划
   - 路线与成本优化
   - 新能源车辆运营管理

4. **经营/数据增值**
   - 车辆、司机、任务和车队画像
   - 经营分析与推荐
   - 数据服务和联合建模

5. **交付保障**
   - 项目管理
   - 硬件接入与交付
   - SaaS 配置与运营服务

### 安全风控闭环能力

历史安全风控材料显示，车队安全能力应按以下闭环判断成熟度：

- **感知层**：ADAS、DMS、BSD、部标主机、视频、轨迹、设备状态和云端补偿。
- **治理层**：设备自检、摄像头角度异常识别、告警过滤、告警召回、误报剔除、告警准确率治理。
- **模型层**：风险因子、风险频率、风险影响、司机/车辆/线路画像、风险等级、未来风险预测。
- **干预层**：车机提醒、TTS/语音提醒、语音对讲、电话外呼、人工护航、客服质检。
- **整改层**：安全培训、专项治理、奖惩激励、事故复盘、车队/司机运营。
- **结果层**：事故率、赔付率、重大/人伤事故、风险消除、设备健康度和客户安全管理质量。

在竞品分析中，应区分硬件能力、平台能力、算法能力和持续运营能力。

- 车辆/设备接入与质量治理
- 在线监测和风险预警
- 省市县多级监管协同
- 企业安全主体责任闭环
- 考核、申诉、处置与决策支撑
- 监管运营中心和常态化服务机制
- 北斗服务商备案/授权与设备接入管理
- 企业、车辆、司机三类安全画像
- 五率考核、安全风险排名和考核通报
- 安全分析报告与企业整改支撑

## 4. 保险产品

- 保前风险因子查询与评分
- 差异化核保辅助
- 保中车辆安全托管
- 7×24 人工干预
- 企业和司机风险画像
- 事故证据与智能反欺诈
- 个性化安全运营周报

### 保险产品链路理解

- **保前核保**：风险因子查询、车辆风险评分、驾驶行为数据、道路/线路风险、承保偏好辅助。
- **保中托管**：实时监控、超速/超时/疲劳等告警、7x24 主动干预、日报/周报/月报、企业/车辆/司机画像。
- **保后理赔**：轨迹与视频证据、疑似事故识别、事故还原、责任判断辅助、损失预估和反欺诈规则。

## 5. 数据与算法底盘

- 货运地图与货车专题数据
- 车辆轨迹、任务、司机、车辆、能耗、路桥、风险数据
- 路线、成本、时效、风险和调度模型
- AIoT 接入和设备数据治理
- 智能调度算法
- 视觉识别与驾驶行为算法
- 数据覆盖、数据增值和 AI 模型支撑
- 技术架构底盘与测试公共能力

### 与丰图历史能力的关系

- **可作为丰行直接相关底盘**：货运地图、货车专题数据、路径/时效/成本/风险模型、轨迹处理、AIoT 车辆接入、驾驶行为算法。
- **可作为潜在支撑**：高速事件巡检、道路病害识别、安全地图、道路隐患排查、施工合规感知、道路资产更新。
- **需谨慎隔离**：通用 GIS、智慧城市、城市数字孪生、政务地理信息、非货运道路资产管理。

## 6. 能力边界提示

上述矩阵同时包含当前规划与历史能力积累。Codex 在对外分析时：

- 可用于判断关联性；
- 不可未经确认写成“丰行已全面商用/已具备成熟产品”；
- 涉及具体版本、客户数量、收入、覆盖车辆数、准确率等必须另行核验。
